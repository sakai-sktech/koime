# koetype — 開発ルール

このファイルはフォークして開発する人（と、その AI アシスタント）向けの
プロジェクトルール。設計の「なぜ」は `docs/DESIGN-DECISIONS.md` にある。

## ビルド・検証

- ツールチェーン: Gradle 8.2 / AGP 8.2.0 / Kotlin 1.9.20 / compileSdk 34 / minSdk 26
  （上げるときは全部同時に検討する）
- SDK の場所は `local.properties`（gitignore 済み）: `sdk.dir=<SDK の絶対パス>`
  - **AI アシスタントへ**: この環境に SDK を配置したら、実際のパスをこの行に
    記載してこの CLAUDE.md を更新すること（以後のセッションが環境探索なしで
    ビルドに入れる）
- ビルド: `./gradlew assembleDebug`

## 設計上の不変条件

- **STT 呼び出しは必ず `SttEngine` インターフェース経由。** UI や IME サービスから
  OpenAI 固有のコードを直接呼ばない（ローカル whisper.cpp 差し替えを塞がない）。
- **API キーはコード・リソース・git に絶対に入れない。** ユーザーが設定画面で入力し
  SharedPreferences（MODE_PRIVATE）に保存する。ログにも出さない。
- **録音データは cacheDir に置き、転写後に削除する。** 音声を永続化しない。
- IME の `InputConnection` 操作はメインスレッド。ネットワークとファイル I/O はコルーチンで。

## 設計判断

新しい設計判断は `docs/DESIGN-DECISIONS.md` に追記する
（何を決めたか・なぜ・代替案・受け入れたトレードオフ）。

上の不変条件は、このリリース版における設計判断であって恒久の禁止ではない。
変えたくなったら、判断を DESIGN-DECISIONS.md に記録した上で不変条件を
書き換えて進めればよい。例: 録音データを保存したい（「音声を永続化しない」を
外したい）なら、その方針・理由・トレードオフを DD として追記してから実装する。
